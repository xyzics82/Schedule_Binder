# Life Binder 로컬 앱 개발 기준 문서

작성일: 2026-05-01  
문서 목적: AI 개발 에이전트에게 이 문서를 입력했을 때, 현재 Life Binder 앱과 최대한 동일한 로컬 작업용 앱을 다시 만들 수 있도록 하기 위한 기준 문서  
문서 범위: 앱의 목적, 화면 구조, 탭별 기능, 일정/체크항목 동작, 시각 디자인, 로컬 사용 흐름  
제외 범위: 사용자 계정, 회원가입, 원격 데이터 저장, 인터넷 배포, 서버 구성, 데이터베이스 설정, 실제 코드 작성 내용

## 1. 제품 한 줄 정의

Life Binder는 계획한 시간과 실제 수행한 시간을 같은 시간표 위에서 비교하고, 주간/오늘/월간 관점으로 체크항목과 일정을 관리하는 개인 자기경영 캘린더 앱이다.

앱은 로컬 폴더 안에서 실행되는 개인용 도구로 만든다. 사용자는 앱을 열자마자 실제 작업 화면에 들어가며, 별도의 계정 화면이나 온라인 연결 절차 없이 자신의 계획과 실행을 관리한다.

## 2. 로컬 앱 전제

이 문서는 인터넷 공개 서비스가 아니라, 사용자의 컴퓨터 안에 있는 폴더에서 작업할 수 있는 앱을 기준으로 한다.

기본 전제:

- 앱은 로컬 폴더 안의 파일들로 구성된다.
- 첫 화면은 바로 앱의 작업 화면이다.
- 사용자 계정 화면은 만들지 않는다.
- 여러 사용자 구분은 고려하지 않는다.
- 외부 서버 연결은 고려하지 않는다.
- 앱 업데이트와 사용 데이터 이전은 별도 운영 절차로 다룬다.
- 이 문서는 배포 방법보다 “어떤 앱을 만들어야 하는가”에 집중한다.

로컬 사용자가 기대하는 경험:

- 폴더 안의 앱 파일을 열면 곧바로 사용할 수 있다.
- 주간 계획, 오늘 실행, 월간 흐름을 한 앱에서 오갈 수 있다.
- 계획 일정과 실행 일정이 시각적으로 구분된다.
- 체크항목은 업무/개인/일일 단위로 빠르게 작성하고 정리할 수 있다.
- 화면은 조밀하지만 답답하지 않아야 한다.

## 3. 제품 철학

Life Binder는 단순한 캘린더가 아니라 자기경영 보드다. 핵심은 “계획과 실행의 차이를 볼 수 있게 하는 것”이다.

앱은 사용자가 다음 질문에 답할 수 있게 도와야 한다.

- 이번 주에 무엇을 하기로 했는가?
- 오늘 실제로 무엇을 했는가?
- 계획한 시간과 실제 수행 시간이 얼마나 달랐는가?
- 중요한 체크항목은 진행되고 있는가?
- 월간 흐름에서 어떤 주가 바빴고, 어떤 날에 실행이 몰렸는가?
- 다음 주 또는 다음 날에 잊지 말아야 할 것은 무엇인가?

사용성 원칙:

- 첫 화면은 설명 페이지가 아니라 실제 작업 화면이어야 한다.
- 앱은 매일 반복해서 쓰는 도구처럼 빠르게 동작해야 한다.
- 계획 작성과 실행 기록은 같은 시간표 위에서 이루어져야 한다.
- 상태 표시는 글자보다 아이콘과 위치로 빠르게 파악되어야 한다.
- 기능 설명 문구가 화면을 차지하지 않게 하고, 필요한 설명은 툴팁이나 작은 안내로 처리한다.

## 4. 대상 사용자

### 4.1 개인 업무 관리 사용자

업무 일정, 개인 목표, 회고를 한 곳에서 관리하려는 사용자다. 여러 프로젝트를 동시에 진행하고, 계획한 시간과 실제 쓴 시간을 비교하고 싶어 한다.

필요한 기능:

- 주간 단위의 업무/개인 체크항목 관리
- 날짜별 시간 블록 계획
- 실제 수행 시간 기록
- 실행 결과 메모
- 월간 흐름 확인

### 4.2 자기계발 사용자

학습, 운동, 독서, 신앙, 가정, 봉사 등 반복적인 루틴을 관리하려는 사용자다. 하루의 실행과 주간의 흐름을 함께 보고 싶어 한다.

필요한 기능:

- 일일 체크항목
- 주간 체크항목
- 오늘의 핵심 행동 정리
- 실행 시간 기록
- 회고와 다음 행동 정리

### 4.3 프리랜서 또는 1인 사업자

프로젝트, 고객 업무, 개인 성장, 행정 업무를 분리해서 관리하려는 사용자다.

필요한 기능:

- 업무 카테고리별 색상 구분
- 프로젝트와 목표 관리
- 실제 실행 시간 추적
- 월간 단위 요약
- 기록과 자료를 모아두는 서브 바인더

## 5. 전체 앱 구조

앱은 왼쪽 사이드바와 오른쪽 메인 작업 영역으로 구성된다.

왼쪽 사이드바에는 앱 이름과 주요 탭이 있다. 사이드바는 좁고 실용적인 폭을 유지한다. 넓은 메뉴판이 아니라 작업 도구의 목차처럼 보여야 한다.

오른쪽 메인 영역은 상단 바와 작업 영역으로 나뉜다.

상단 바에는 현재 탭 제목, 날짜 이동 버튼, 오늘 버튼, 필요한 경우 Plan/Do 모드 버튼, 내보내기 버튼, 현재 화면에서 중요한 보조 입력 영역이 배치된다.

작업 영역에는 선택한 탭의 실제 내용이 표시된다.

주요 탭:

- 주간
- 오늘
- 월간
- 목표
- 프로젝트
- 서브 바인더
- 리뷰
- 통계
- 가이드

탭 선택 시 규칙:

- 선택된 탭은 색상과 강조선으로 명확히 표시한다.
- 탭 왼쪽에 불필요한 장식 점은 두지 않는다.
- 각 탭은 첫 화면에서 바로 사용할 수 있는 작업 화면이어야 한다.
- 설명성 큰 문단은 배치하지 않는다.

## 6. 공통 화면 레이아웃

### 6.1 왼쪽 사이드바

사이드바 구성:

- 앱 이름: Life Binder
- 짧은 부제: 목표, 시간, 기록을 연결하는 자기경영 보드
- 탭 목록
- 하단의 보조 버튼 영역

사이드바 디자인:

- 폭은 좁게 유지한다.
- 글자는 작지만 읽기 쉬워야 한다.
- 각 탭은 한 줄로 정렬한다.
- 탭에는 짧은 아이콘 또는 알파벳 표시와 한글 이름을 함께 둔다.
- 현재 탭은 배경색과 글자색으로 구분한다.
- 과도한 그림 장식은 사용하지 않는다.

주간 탭 사용법 안내:

- 주간 작성 방법은 주간 탭 항목에 커서를 오래 올렸을 때 툴팁으로 보여준다.
- 주간 일정 화면 안에는 긴 작성 방법 안내문을 넣지 않는다.
- 툴팁은 짧고 실용적으로 작성한다.

### 6.2 상단 바

상단 바는 현재 화면의 맥락을 보여주고, 자주 쓰는 이동 버튼을 제공한다.

공통 요소:

- 현재 탭 제목
- 현재 기준 날짜 또는 기간
- 이전/다음 이동 버튼
- 오늘 버튼
- 필요한 경우 내보내기 버튼

오늘 버튼:

- 누르면 기준 날짜가 실제 오늘 날짜로 이동한다.
- 주간 탭에서는 오늘이 포함된 주간으로 이동한다.
- 오늘 탭에서는 오늘 날짜로 이동한다.
- 월간 탭에서는 오늘이 포함된 월로 이동한다.

Don't Forget 영역:

- 각 주요 탭의 상단 중앙 근처에 배치한다.
- 사용자가 잊지 말아야 할 짧은 메모를 적는 영역이다.
- “Don't Forget” 제목은 다른 보조 제목보다 조금 더 크게 보인다.
- 항목은 간단히 추가, 수정, 삭제할 수 있어야 한다.
- 체크상태 설명은 이 영역 아래가 아니라 주간 체크항목의 “업무” 제목 옆에 작게 배치한다.

## 7. 핵심 데이터 개념

이 문서는 실제 저장 방식이나 파일 형식을 지정하지 않는다. 다만 앱 내부에서 다음 개념을 일관되게 다루어야 한다.

### 7.1 날짜

날짜는 모든 일정과 체크항목의 기준이다.

날짜 표시 규칙:

- 주간/오늘 화면의 날짜는 “4. 20. (월)” 형태로 한 줄에 표시한다.
- 요일과 날짜를 위아래로 분리하지 않는다.
- 주간 화면에서는 각 날짜 칼럼 상단에 날짜와 요일을 함께 보여준다.
- 오늘 탭 제목 옆에도 오늘 날짜와 요일을 보여준다.

### 7.2 주간

주간은 월요일부터 일요일까지의 7일 단위다.

주간 화면은 사용자가 한 주의 계획과 실행을 동시에 관리하는 핵심 화면이다.

주간 데이터 개념:

- 주간 체크항목
- 날짜별 하루 체크항목
- 날짜별 계획 일정
- 날짜별 실행 일정
- Don't Forget 메모
- 주간 시간 배분 요약

### 7.3 일정

일정은 특정 날짜의 시작 시간과 끝 시간을 가진 시간 블록이다.

일정의 종류:

- 계획 일정: 사용자가 원래 하기로 계획한 일정
- 실행 일정: 실제로 수행한 일정

일정의 필수 속성:

- 제목
- 날짜
- 시작 시간
- 끝 시간
- 카테고리
- 종류
- 상세 메모

일정의 표시 원칙:

- 시간표 위에 실제 시간 범위만큼 세로 길이로 표시한다.
- 일정 제목은 가능한 한 모두 표시한다.
- 공간이 부족한 경우에만 말줄임 표시를 사용한다.
- 일정 제목 때문에 그래픽이 시간 범위를 넘어서면 글자 크기를 줄인다.
- 시간 텍스트는 일정 제목과 비슷한 크기와 굵기로 표시한다.
- 시간 텍스트만 지나치게 두껍게 만들지 않는다.

### 7.4 체크항목

체크항목은 일정이 아니라 해야 할 일, 확인할 일, 진행 상태를 기록하는 작은 항목이다.

체크항목의 종류:

- 주간 업무 체크항목
- 주간 개인 체크항목
- 하루 체크항목
- 월간 날짜별 체크항목

체크항목의 필수 속성:

- 내용
- 상태
- 소속 날짜 또는 주간
- 업무/개인 구분

체크항목 디자인 원칙:

- 체크 버튼은 작고 깔끔한 아이콘형 버튼으로 만든다.
- 체크 버튼 내부에 불필요한 작은 원형 장식을 넣지 않는다.
- 체크 버튼과 텍스트 입력칸은 세로 중앙이 정확히 맞아야 한다.
- 글씨 아래가 잘리지 않아야 한다.
- 위아래 간격은 타이트하게 유지한다.
- 여러 항목이 있어도 전체 레이아웃이 아래로 과하게 밀리지 않게 한다.

## 8. 체크항목 상태 규칙

체크항목 상태는 글자가 아니라 아이콘 중심으로 표시한다.

상태 종류:

- 미정
- 완료
- 진행중
- 취소
- 연기/위임

상태 표시:

- 미정: 비어 있는 작은 사각형 또는 얇은 테두리 아이콘
- 완료: 체크 표시 아이콘
- 진행중: 오른쪽 방향 화살표 아이콘
- 취소: X 표시 아이콘
- 연기/위임: 서로 마주보거나 방향 전환을 암시하는 화살표 아이콘

표시 규칙:

- 체크항목 자체에는 상태 설명 글자를 넣지 않는다.
- 상태 아이콘은 선이 깨지지 않고 선명해야 한다.
- 모든 상태 아이콘은 같은 크기와 같은 중심축을 가진다.
- 아이콘 버튼은 너무 크지 않게 한다.
- 사용자가 상태 버튼을 누르면 상태가 순환되거나 선택 메뉴가 열린다.

상태 설명 위치:

- 주간 체크항목 영역의 “업무” 제목 옆에 작은 범례로 표시한다.
- 범례는 너무 작지 않게 하고, 실제 사용자가 알아볼 수 있어야 한다.
- 범례는 화면 중앙으로 어색하게 떨어지지 않고, 제목과 자연스럽게 같은 줄에 놓인다.
- 범례에도 텍스트는 최소화하되 필요한 설명은 읽을 수 있어야 한다.

## 9. 시간표 공통 규칙

시간표는 일정 앱의 중심이다.

시간 범위:

- 기본적으로 새벽 5시부터 밤 12시까지를 표시한다.
- 필요하면 더 넓은 시간 범위를 지원할 수 있지만, 기본 화면은 5시 이후 일과를 보기 좋게 한다.

시간 축:

- 왼쪽에는 시간 눈금이 있다.
- 각 시간 눈금은 일정 블록의 실제 위치와 정확히 맞아야 한다.
- 시간과 일정 그래픽이 어긋나면 안 된다.

스크롤:

- 시간표는 세로로 스크롤할 수 있다.
- 화면을 아래로 내렸을 때 일정 그래픽이 상단 영역을 침범하면 안 된다.
- 일정이 상단 바나 체크항목 영역 위로 겹쳐 보이면 안 된다.
- 가로 스크롤은 시간표 아래뿐 아니라 5시 시간선 위쪽에도 제공한다.
- 사용자가 아래까지 내리지 않아도 좌우 날짜 이동을 할 수 있어야 한다.

일정 블록:

- 계획 일정과 실행 일정은 모두 시간표 위에 표시한다.
- 일정 블록은 카테고리 색상으로 구분한다.
- 블록 테두리는 또렷하지만 과도하게 두껍지 않게 한다.
- 작은 일정도 최소한의 높이를 가져 클릭 가능해야 한다.
- 너무 짧은 일정에서는 텍스트가 넘치지 않게 자동으로 줄이거나 말줄임 처리한다.

겹침 표시:

- 계획 일정과 실행 일정이 같은 시간대에 있으면 나란히 떨어뜨리지 않고 약간 겹쳐 보이게 한다.
- Plan 모드와 Do 모드의 겹침 정도는 동일해야 한다.
- 실행 일정이 다른 계획 일정 시간대를 침범해도 기존 계획 일정의 크기가 변하면 안 된다.
- 겹침은 “뒤로 들어간 것처럼” 보이되, 제목과 시간 확인이 가능해야 한다.
- 겹친 블록은 클릭 가능한 우선순위가 자연스럽게 유지되어야 한다.

## 10. Plan 모드와 Do 모드

앱의 핵심 모드는 Plan과 Do다.

Plan 모드:

- 계획 일정을 작성하고 수정하는 모드다.
- 계획 블록이 주된 작업 대상이다.
- 실행 일정도 배경 또는 겹침 형태로 참고할 수 있다.
- 실행 일정의 시작 시간과 끝 시간은 일정 블록의 앞부분과 끝부분 위치에 맞게 표시한다.

Do 모드:

- 실제 수행 일정을 작성하고 수정하는 모드다.
- 계획 일정은 참고용으로 보인다.
- 계획 일정을 클릭하면 별도 팝업을 먼저 띄우지 않고 바로 실행 일정이 만들어진다.
- 만들어진 실행 일정은 사용자가 필요할 때 직접 클릭해 세부 수정한다.
- 실행 일정이 계획보다 길어지거나 다른 계획을 침범해도 계획 일정은 줄어들지 않는다.

모드 버튼 위치:

- Plan/Do 전환 버튼은 주간 탭과 오늘 탭에 둔다.
- 월간, 목표, 프로젝트, 리뷰 등에는 불필요하게 표시하지 않는다.
- 주간 화면에서는 각 날짜 헤더 옆에 자연스럽게 배치한다.
- 오늘 화면에서는 오늘 제목과 날짜 옆에 배치한다.

Plan 복사:

- 하루의 계획 일정을 다른 날로 복사할 수 있어야 한다.
- “Plan 복사” 버튼은 날짜 헤더 근처에 둔다.
- 복사 대상 날짜를 선택할 수 있어야 한다.
- 복사 시 하루 체크항목까지 복사할지, 계획 일정만 복사할지 선택할 수 있으면 좋다.
- 기본 동작은 계획 일정 복사다.

## 11. 카테고리 규칙

일정은 카테고리를 가진다.

기본 카테고리 예시:

- 주업무
- 보조업무
- 신앙/가정/봉사
- 건강
- 학습
- 휴식
- 이동
- 기타

카테고리 디자인:

- 각 카테고리는 고유한 색상을 가진다.
- 색상은 서로 구분되어야 한다.
- 한 가지 색 계열만으로 화면이 단조롭게 보이지 않게 한다.
- 어두운 배경 위에서도 텍스트가 읽혀야 한다.

카테고리 수정:

- 일정 세부 팝업에서 카테고리를 수정할 수 있다.
- Do 모드에서도 실행 일정의 카테고리를 바꿀 수 있다.
- 카테고리가 바뀌면 일정 블록 색상이 즉시 반영된다.

## 12. 주간 탭 상세 명세

주간 탭은 앱의 중심 화면이다.

### 12.1 주간 상단

표시 요소:

- 제목: 주간
- 기간: “4월 19일 (일)부터 4월 25일 (토)까지”처럼 표시
- 이전 주 버튼
- 다음 주 버튼
- 오늘 버튼
- 필요한 경우 내보내기 버튼
- Don't Forget 영역

이번 주 체크박스에 “2026-04-19 주간” 같은 주간 날짜 표기는 표시하지 않는다.

### 12.2 주간 체크항목 영역

주간 체크항목은 업무와 개인으로 나뉜다.

구성:

- 왼쪽 또는 위쪽에 “업무” 영역
- 오른쪽 또는 아래쪽에 “개인” 영역
- 두 영역 사이에는 시각적으로 구분되는 중앙 라인을 둔다.
- 각 영역에 체크항목 입력칸과 추가 버튼을 둔다.

디자인:

- 항목 왼쪽의 불필요한 점 표시를 제거한다.
- 위아래 간격을 일일 체크항목처럼 타이트하게 한다.
- 체크 버튼과 텍스트 입력칸의 수평 정렬을 정확히 맞춘다.
- 글씨가 입력칸 아래로 잘리지 않게 한다.
- 추가 버튼은 작고 단정한 디자인으로 만든다.
- 추가 버튼은 입력칸과 세로 중앙이 맞아야 한다.

체크항목 입력:

- 항목 내용을 직접 입력할 수 있다.
- 상태 버튼을 눌러 미정, 완료, 진행중, 취소, 연기/위임 상태를 바꿀 수 있다.
- 삭제 버튼은 작게 배치한다.
- 화면에 너무 많은 높이를 차지하지 않게 한다.

### 12.3 주간 날짜 칼럼

한 주의 각 날짜는 가로 칼럼으로 표시된다.

각 날짜 헤더:

- 날짜와 요일은 한 줄로 표시한다.
- 예: “4. 20. (월)”
- 날짜 글자는 요일과 비슷하게 크고 읽기 쉬워야 한다.
- Plan/Do 버튼과 Plan 복사 버튼은 날짜 헤더 옆에 정렬한다.

각 날짜 칼럼 구성:

- 날짜 헤더
- 하루 체크항목 입력 영역
- 시간표
- 계획/실행 일정 블록

날짜 칼럼 너비:

- Plan 모드에서는 실행 일정 영역이 과도하게 넓지 않아야 한다.
- Do 모드에서는 계획 일정 영역이 과도하게 넓지 않아야 한다.
- 보조로 표시되는 일정 영역은 시간 글자가 들어갈 정도의 폭이면 충분하다.
- 사용자가 주된 작업 대상과 참고 일정을 빠르게 구분할 수 있어야 한다.

### 12.4 하루 체크항목 영역

각 날짜마다 하루 체크항목 영역이 있다.

기본 규칙:

- 입력칸과 추가 버튼을 날짜 헤더 아래에 둔다.
- 체크항목은 기본적으로 3개 공간을 확보한다.
- 실제 항목이 0개여도 3개가 들어갈 수 있는 일정한 공간이 유지된다.
- 항목이 3개 이하이면 그 안에서 정렬된다.
- 항목이 4개 이상이면 기본 화면에는 3개까지만 보이고, 마우스를 올리면 전체 항목을 볼 수 있다.
- 항목 추가 시 시간표가 아래로 밀려 시간 위치가 어긋나면 안 된다.

디자인:

- 위아래 간격은 아주 타이트하게 한다.
- 체크 버튼은 작고 선명해야 한다.
- 입력칸과 버튼의 중앙 정렬이 정확해야 한다.
- 추가 버튼은 작고 귀여운 느낌의 단정한 버튼으로 만든다.
- 삭제 버튼은 항목 오른쪽 끝에 작게 둔다.

스크롤 유지:

- 화면이 작은 경우 체크항목을 입력해도 사용자의 스크롤 위치가 처음으로 돌아가면 안 된다.
- 입력 직후에도 사용자가 보던 위치가 유지되어야 한다.

### 12.5 주간 시간표

주간 시간표는 7일을 가로로 나열하고, 각 날짜의 시간 흐름을 세로로 보여준다.

필수 동작:

- 좌우로 스크롤 가능
- 위쪽과 아래쪽 모두에서 가로 스크롤 가능
- 세로 스크롤 시 상단 체크항목 영역을 침범하지 않음
- 일정 블록의 위치가 시간 눈금과 정확히 맞음
- 계획과 실행 일정의 겹침 정도가 Plan/Do 모드에서 동일함

## 13. 오늘 탭 상세 명세

오늘 탭은 선택한 하루를 집중해서 보는 화면이다.

### 13.1 오늘 상단

표시 요소:

- 제목: 오늘
- 제목 옆 날짜와 요일
- 예: “오늘 4. 20. (월)”
- 이전 날 버튼
- 다음 날 버튼
- 오늘 버튼
- Plan/Do 모드 버튼
- Don't Forget 영역

제거할 요소:

- “오늘의 주간일정” 같은 설명성 제목
- 그 아래의 안내 문장
- 불필요한 큰 설명 영역

### 13.2 오늘 체크항목

오늘 탭에서는 해당 날짜의 하루 체크항목이 모두 잘 보여야 한다.

필수 동작:

- 특정 날짜에 체크항목이 여러 개 있어도 위쪽과 아래쪽으로 갈라져 보이면 안 된다.
- 3개만 보이고 나머지가 사라지면 안 된다.
- 항목은 한 영역 안에서 순서대로 정렬되어야 한다.
- 체크 버튼과 텍스트가 정확히 수평 정렬되어야 한다.
- 글씨가 잘리지 않아야 한다.

오늘 탭의 체크항목은 주간 탭의 해당 날짜 하루 체크항목과 같은 데이터를 바라본다.

주간 탭에서 수정하면 오늘 탭에도 반영되고, 오늘 탭에서 수정하면 주간 탭에도 반영된다.

### 13.3 오늘 시간표

오늘 시간표는 하루 전체를 넓게 보여준다.

Plan/Do 동작:

- 오늘 탭에서도 계획 일정과 실행 일정을 모두 수정할 수 있다.
- 주간 탭에서 하는 것과 동일한 방식으로 일정 제목, 시간, 카테고리, 상세 내용을 수정할 수 있다.
- 오늘 탭에서 수정한 내용은 주간 탭에도 즉시 반영된다.
- 오늘 탭에서 계획 일정을 클릭해 실행 일정을 만들 수 있다.

## 14. 월간 탭 상세 명세

월간 탭은 한 달의 흐름과 주별 요약을 보는 화면이다.

디자인 방향:

- 주간 탭의 어두운 자기경영 보드 느낌을 유지한다.
- 월간 화면은 단순한 달력표가 아니라 “월간 흐름판”처럼 보여야 한다.
- 각 주의 흐름과 날짜별 체크항목을 함께 파악할 수 있어야 한다.

### 14.1 월간 상단

표시 요소:

- 제목: 월간
- 현재 월
- 이전 월 버튼
- 다음 월 버튼
- 오늘 버튼
- Don't Forget 영역

제거할 요소:

- 오른쪽 체크박스 패널
- “이번달 프로젝트” 패널
- 긴 설명 문장

### 14.2 월간 그리드

월간은 주 단위 행과 요일 단위 열로 구성한다.

각 행:

- 왼쪽에 주간 요약 칸을 둔다.
- 오른쪽에는 월~일 날짜 칸을 둔다.

날짜 칸:

- 날짜 숫자
- 그날의 대표 체크항목
- 대표 일정 개수 또는 요약
- 필요하면 작은 색상 표시

해당 월과 전후 월 구분:

- 현재 보고 있는 월에 속한 날짜 칸은 조금 더 밝게 표시한다.
- 이전 달 또는 다음 달에 속한 날짜 칸은 흐리게 표시한다.
- 흐린 날짜도 주간 요약 계산에는 포함한다.
- 월간 화면은 달력의 빈칸을 만들지 않고, 실제 주간 흐름을 유지한다.

### 14.3 주간 요약 행

각 주의 왼쪽에는 그 주를 설명하는 요약 칸이 있다.

요약 내용:

- 그 주의 주요 계획
- 실행이 많았던 카테고리
- 체크항목 완료 흐름
- 사용자가 직접 입력한 한 줄 메모

전후 월 날짜가 포함된 주라도 요약에는 함께 반영한다.

### 14.4 월간 날짜 클릭

월간 날짜 칸을 클릭하면 그 날짜의 체크항목을 관리할 수 있다.

가능한 작업:

- 체크항목 추가
- 체크항목 수정
- 체크항목 삭제
- 상태 변경
- 해당 날짜의 주요 일정 확인

체크항목이 많은 날:

- 날짜 칸 안에는 대표 항목만 표시한다.
- 다 표시되지 않는 경우 마우스를 올리면 전체 항목을 볼 수 있다.
- 툴팁 또는 작은 팝오버는 날짜 칸을 가리지 않게 배치한다.

## 15. 목표 탭 상세 명세

목표 탭은 장기 목표와 월간/주간 실행을 연결하는 공간이다.

구성:

- 연간 목표
- 월간 목표
- 주간 핵심 목표
- 목표별 진행 상태
- 목표 관련 메모

디자인:

- 목표는 카드나 행 단위로 정리한다.
- 장식적 랜딩 페이지처럼 만들지 않는다.
- 목표 제목, 진행률, 다음 행동이 빠르게 보이게 한다.
- 체크항목 상태 아이콘과 동일한 시각 언어를 사용할 수 있다.

기능:

- 목표 추가
- 목표 수정
- 목표 삭제
- 목표 상태 변경
- 목표와 관련된 주간 체크항목 연결

## 16. 프로젝트 탭 상세 명세

프로젝트 탭은 여러 업무나 개인 프로젝트를 모아보는 공간이다.

구성:

- 프로젝트 목록
- 프로젝트별 상태
- 다음 행동
- 관련 일정
- 관련 메모

프로젝트 상태 예시:

- 진행 전
- 진행중
- 대기
- 완료
- 취소

디자인:

- 업무 도구처럼 조밀하고 정리된 표 또는 리스트 형태가 적합하다.
- 프로젝트마다 색상 라벨을 줄 수 있다.
- 제목, 마감, 다음 행동이 한눈에 보여야 한다.

## 17. 서브 바인더 탭 상세 명세

서브 바인더는 자료와 생각을 모아두는 공간이다.

구성:

- 카테고리별 노트
- 짧은 메모
- 참고 링크 제목
- 자료 설명

디자인:

- 실제 문서함이나 바인더처럼 정돈된 느낌을 준다.
- 검색 또는 필터가 있으면 좋다.
- 내용 작성 영역은 충분한 높이를 제공한다.

## 18. 리뷰 탭 상세 명세

리뷰 탭은 사용자가 자신의 계획과 실행을 돌아보는 공간이다.

구성:

- 일일 리뷰
- 주간 리뷰
- 월간 리뷰
- 잘한 점
- 아쉬운 점
- 다음 행동

주요 질문:

- 계획대로 한 일은 무엇인가?
- 계획과 달라진 일은 무엇인가?
- 시간을 많이 쓴 영역은 무엇인가?
- 다음 주에 조정할 것은 무엇인가?

디자인:

- 글을 쓰기 편한 넓은 입력 공간을 제공한다.
- 일정과 체크항목을 참고하면서 작성할 수 있어야 한다.
- 리뷰는 과도한 장식보다 읽기 쉬운 문서형 구성이 좋다.

## 19. 통계 탭 상세 명세

통계 탭은 시간 사용과 카테고리 흐름을 요약한다.

구성:

- 이번 주 계획 시간 합계
- 이번 주 실행 시간 합계
- 카테고리별 시간 분포
- 계획 대비 실행 차이
- 체크항목 상태 요약

제거할 요소:

- 필요 없는 “일 완료율” 탭
- 필요 없는 “시간 블록” 란

디자인:

- 그래프는 작고 선명해야 한다.
- 색상은 일정 카테고리 색상과 연결된다.
- 숫자와 막대가 실제 시간 합계와 맞아야 한다.

## 20. 가이드 탭 상세 명세

가이드 탭은 앱 사용법을 짧게 정리하는 보조 화면이다.

구성:

- Plan 모드 사용법
- Do 모드 사용법
- 체크항목 상태 의미
- 주간 계획 작성 흐름
- 오늘 실행 기록 흐름
- 월간 검토 흐름

디자인:

- 긴 설명서처럼 보이지 않게 한다.
- 짧은 문장과 작은 예시 중심으로 구성한다.
- 자주 쓰는 기능 위주로 설명한다.

## 21. 일정 작성/수정 팝업 명세

일정을 클릭하거나 새 일정을 만들 때 일정 편집 팝업을 표시한다.

팝업 구성:

- 일정 제목
- 시작 시간
- 끝 시간
- 카테고리
- 종류
- 상세 메모
- 저장 버튼
- 삭제 버튼
- 닫기 버튼

시간 입력:

- 사용자가 직접 시작 시간과 끝 시간을 입력할 수 있어야 한다.
- 시간 입력은 빠르게 고칠 수 있어야 한다.
- 저장하면 시간표의 그래픽 위치와 높이가 수정된 시간에 맞게 바뀐다.

상세 메모:

- 여러 줄 입력을 허용한다.
- 사용자가 엔터로 줄을 나누어 작성하면 일정 블록 또는 상세 보기에서도 줄바꿈이 유지되어야 한다.
- 일정 블록 공간이 충분하면 여러 줄 내용을 가능한 한 보여준다.
- 공간이 부족할 때만 말줄임 처리한다.

Do 모드에서 계획 일정 클릭:

- 계획 일정을 클릭하면 바로 실행 일정이 만들어진다.
- 팝업은 자동으로 먼저 뜨지 않는다.
- 실행 일정을 만든 후 사용자가 그 블록을 클릭하면 세부 내용을 수정할 수 있다.

## 22. 시각 디자인 기준

전체 분위기:

- 어두운 청록색 계열의 작업 보드 느낌
- 밝은 포인트 컬러
- 조밀하지만 정돈된 배치
- 반복 사용에 피로하지 않은 색상

색상 원칙:

- 배경은 어두운 청록색 또는 남색 계열을 사용한다.
- 카테고리는 서로 다른 색상으로 구분한다.
- 한 가지 색만 반복되는 단조로운 화면을 피한다.
- 강조 버튼은 민트, 핑크, 노랑 등 서로 구별되는 포인트를 사용한다.

글꼴 원칙:

- 숫자와 시간은 읽기 쉬워야 한다.
- 시간 텍스트는 일정 제목보다 과도하게 두껍지 않게 한다.
- 버튼 안 글씨는 잘리지 않아야 한다.
- 작은 체크항목 글씨도 실제 사용자가 읽을 수 있어야 한다.

버튼:

- Plan/Do 버튼은 작지만 분명해야 한다.
- 추가 버튼은 크기를 줄이고 정렬을 맞춘다.
- 삭제 버튼은 항목 오른쪽에 작게 배치한다.
- 아이콘 버튼에는 필요한 경우 툴팁을 제공한다.

카드와 패널:

- 큰 화면 전체를 카드 안에 다시 넣는 구조는 피한다.
- 반복 항목, 편집 팝업, 작은 정보 묶음에만 카드 형태를 사용한다.
- 중첩 카드처럼 보이는 디자인은 피한다.

## 23. 반응형 요구사항

데스크톱:

- 주간 화면은 7일 칼럼을 가로로 배치한다.
- 화면이 좁으면 가로 스크롤로 이동한다.
- 사이드바는 좁은 폭을 유지한다.
- 시간표의 위쪽과 아래쪽 모두에서 가로 이동이 가능해야 한다.

태블릿/작은 화면:

- 탭 메뉴와 작업 영역이 겹치지 않아야 한다.
- 체크항목 입력 후 스크롤 위치가 초기화되지 않아야 한다.
- 버튼 글씨가 잘리지 않아야 한다.
- 일정 블록 텍스트가 시간 범위를 침범하지 않아야 한다.

모바일에 가까운 폭:

- 주간 칼럼은 가로 스크롤로 유지한다.
- 날짜 헤더, 체크항목, 시간표가 서로 겹치지 않아야 한다.
- 팝업은 화면 안에 들어와야 한다.
- 입력칸은 손가락으로 누를 수 있는 최소 크기를 유지한다.

## 24. 로컬 보존 기준

이 문서는 외부 저장소나 여러 기기 동기화를 다루지 않는다.

다만 로컬 앱으로서 다음 경험은 유지하는 것이 좋다.

- 같은 브라우저에서 앱을 다시 열었을 때 최근 작업 상태가 남아 있으면 좋다.
- 사용자가 실수로 새로고침해도 작성 중인 주요 내용이 가능한 한 유지되면 좋다.
- 앱 업데이트 중 기존 로컬 작업 파일을 덮어쓰지 않도록 주의해야 한다.
- 중요한 데이터 이전이나 백업 기능은 후속 과제로 분리한다.

로컬 보존 기능은 앱의 핵심 화면 개발이 끝난 뒤 붙여도 된다.

## 26. AI 에이전트 작업 지시 기준

AI 에이전트에게 이 문서를 전달할 때의 기준:

- 설명 페이지보다 실제 작업 화면을 먼저 만들게 한다.
- 주간 탭을 앱의 중심으로 만든다.
- Plan/Do 모드의 시각 차이를 명확히 만든다.
- 계획과 실행 일정은 같은 시간표 위에서 겹쳐 보이게 만든다.
- 체크항목 정렬 문제를 중요하게 다룬다.
- 작은 화면에서 스크롤 위치가 유지되게 한다.
- 월간 화면은 주간 디자인의 연장선에서 다시 설계한다.
- 로그인, 외부 서버, 인터넷 공개 절차는 만들지 않는다.

AI 에이전트가 특히 주의해야 할 품질 기준:

- 체크 버튼의 세로 중앙 정렬
- 체크항목 글씨 잘림 방지
- 하루 체크항목 3개 기본 공간 확보
- 4개 이상 항목의 hover 전체 보기
- 상단과 하단 가로 스크롤 제공
- 일정 그래픽의 상단 침범 방지
- Plan 모드와 Do 모드의 동일한 겹침 정도
- 일정 제목의 줄바꿈 유지
- 충분한 공간에서는 말줄임을 하지 않음
- 공간 부족 시에만 말줄임 또는 글자 크기 축소

## 27. 완성 판정 기준

다음 조건을 만족하면 로컬 앱 기준의 1차 완성으로 본다.

- 앱을 열면 바로 작업 화면이 나타난다.
- 왼쪽 사이드바에서 모든 탭을 이동할 수 있다.
- 주간 화면에서 월~일 날짜가 한 줄 형식으로 표시된다.
- 주간 업무/개인 체크항목이 중앙 라인으로 구분된다.
- 체크항목 버튼과 텍스트가 정확히 정렬된다.
- 하루 체크항목은 3개 기본 공간을 유지한다.
- 주간 시간표의 일정 위치가 시간 눈금과 맞다.
- Plan 모드와 Do 모드에서 일정 겹침 정도가 동일하다.
- Do 모드에서 계획 일정을 클릭하면 바로 실행 일정이 생성된다.
- 오늘 탭에서 날짜와 요일이 제목 옆에 표시된다.
- 오늘 탭에서 체크항목이 누락 없이 정렬된다.
- 오늘 탭에서 수정한 일정이 주간 탭에도 반영된다.
- 월간 탭에서 현재 월과 전후 월 날짜가 명확히 구분된다.
- 월간 날짜 클릭으로 체크항목을 추가, 수정, 삭제할 수 있다.
- 일정 메모의 줄바꿈이 표시 화면에서도 유지된다.
- 화면을 스크롤해도 일정 그래픽이 상단 영역을 침범하지 않는다.

## 28. 핵심 요약

Life Binder 로컬 앱은 “주간 계획, 오늘 실행, 월간 흐름”을 연결하는 개인 시간 관리 도구다.

가장 중요한 화면은 주간 탭이다. 주간 탭에서는 날짜별 시간표, Plan/Do 모드, 주간 체크항목, 하루 체크항목, Plan 복사가 안정적으로 동작해야 한다.

오늘 탭은 주간의 하루를 확대해 보는 화면이다. 오늘 탭에서 수정한 내용은 주간 화면과 같은 기준으로 움직여야 한다.

월간 탭은 단순한 달력보다 한 달의 흐름과 주별 요약을 보여주는 화면이어야 한다.

이 버전의 기준 문서는 로컬 폴더에서 작업하는 앱을 만들기 위한 문서다. 외부 계정, 원격 저장, 인터넷 공개 절차는 앱 범위에서 제외한다.
