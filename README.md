# Schedule Binder

> 구 명칭 "Life Binder Web" — 2026-06-10에 앱 이름을 Schedule Binder로 확정.

3P Binder형 자기경영 흐름을 참고해 만든 정적 웹앱 MVP입니다. 공식 3P바인더의 속지나 디자인을 복제하지 않고, 목표, 시간, 기록, 리뷰, 지식화를 웹 UI로 재해석했습니다.

## 실행 방법

`index.html` 파일을 브라우저로 열면 바로 사용할 수 있습니다. 별도 설치나 서버가 필요 없습니다.

## 포함 기능

- 오늘 실행판
- 주간 계획판
- 월간 로드맵
- 목표 관리
- 프로젝트 관리
- 할 일 관리
- 서브 바인더 노트
- 일간/주간 리뷰
- 시간 사용 통계
- 계획/실행 라인형 시간표
- 주간 화면 기본 진입
- 독자 설계 종이형 `페이퍼` 주간 시트
- 마우스 드래그 계획 작성 및 실행 덮어쓰기
- 15분 단위 일정 작성과 드래그 시간 수정
- 기존 계획/실행 항목 드래그 이동과 위/아래 핸들 시간 조정
- Google Calendar식 겹침 일정 표시
- `Plan` / `Do` 모드 전환과 계획 클릭 후 실행 기록 작성
- 하루 Plan 전체를 다른 날짜로 복사
- 일정 근처에서 열리는 미니 편집 팝업
- 여러 줄 일정 제목 표시
- 주업무, 보조업무, 신앙/가정/봉사, 자기개발, 휴먼 네트워크 5개 분류와 구분 강화 색상 표시
- Do 일정 카테고리 별도 수정
- 실행 기록 형광 하이라이트
- 업무/개인으로 나뉜 주간 체크박스 작성 및 제목 수정
- 오늘 체크박스와 주간 체크박스 분리
- 오늘 탭의 compact 스케줄 요약
- 스케줄 항목별 연결 메모와 사진 첨부
- 주요 기능 예시 가이드
- JSON 내보내기
- Google 캘린더 양방향 동기화 (Plan 전용)
- 모바일 화면 최적화 (상단 고정 내비게이션, 하루 단위 주간표 스와이프, 하단 시트형 일정 팝업)

데이터는 브라우저 `localStorage`에 저장되고, 로그인 시 Supabase로 원격 동기화됩니다.

## Google 캘린더 연동

Plan(계획) 일정만 전용 "Schedule Binder" 캘린더와 양방향 동기화합니다. 실행(Do) 기록은 앱 안에만 남습니다. 동기화 매핑·충돌 규칙 등 상세 사양은 `GCAL_INTEGRATION_REQUEST.md`를 참고하세요.

### OAuth Client ID 발급 절차

1. [Google Cloud Console](https://console.cloud.google.com/)에 접속해 새 프로젝트를 만듭니다.
2. **API 및 서비스 → 라이브러리**에서 `Google Calendar API`를 검색해 **사용 설정**합니다.
3. **API 및 서비스 → OAuth 동의 화면**에서 앱을 만듭니다.
   - User Type: **외부(External)**
   - 게시 상태가 "테스트"라면 **테스트 사용자**에 본인 Google 계정을 추가합니다.
4. **API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**를 선택합니다.
   - 애플리케이션 유형: **웹 애플리케이션**
   - **승인된 JavaScript 원본**에 앱 주소를 추가합니다.
     - 배포: `https://xyzics82.github.io`
     - 로컬 테스트: `http://localhost:8000`
   - 리디렉션 URI는 필요 없습니다 (GIS 토큰 방식).
5. 생성된 **Client ID**(`....apps.googleusercontent.com`)를 복사합니다.
6. 앱의 **설정 탭**에서 Client ID를 붙여넣고 저장 → **Google 계정 연결**을 누릅니다.

연결하면 "Schedule Binder" 캘린더가 자동 생성되고(기존 "Life Binder" 캘린더가 있으면 자동으로 이름이 변경됨), 이후 일정 저장 시 즉시 GCal로 전송되며 GCal 변경분은 기본 5분 간격(설정에서 변경 가능)으로 가져옵니다. Client ID와 토큰은 이 브라우저의 `localStorage`에만 저장됩니다.
